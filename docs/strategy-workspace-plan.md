# 전략 워크스페이스 기획서

**트렌드 레이더 확장 — "주제를 입력하면 신사업 전략과 시나리오가 한 페이지에서 나오는" 개인 업무 도구**

| 항목 | 내용 |
|---|---|
| 문서 버전 | v1 (2026. 7. 16.) |
| 작성 목적 | 신사업 전략·시나리오 수립 업무를 한 페이지에서 처리하는 도구의 틀(와꾸) 정의 |
| 벤치마크 | 신사업전략팀 AI Strategy Builder (Lovable) |
| 현재 상태 | Phase 1 구현 완료 (`strategy.html`, 브랜치 `claude/strategy-page-entry-itr82l`) |

## 1. 배경과 문제의식

지금 업무 흐름은 두 토막으로 끊겨 있다.

- **근거 수집**은 트렌드 레이더로 해결된다 — 키워드 하나로 뉴스·유튜브·블로그/카페 반응, 언급 추이, 빈출 단어까지 나온다.
- 그러나 **전략 문서화**는 수작업이다 — 수집한 근거를 들고 ChatGPT/Claude를 오가며 시장조사 따로, 경쟁 분석 따로 묻고, 답변을 복사해 문서로 재조립한다. 단계 간 맥락이 끊기고, 이전 버전은 사라지고, 보고서 양식은 매번 새로 만든다.

신사업전략팀의 AI Strategy Builder처럼 **"주제와 맥락을 입력하면 5단계 산출물과 최종 리포트가 한 페이지에서 나오는"** 구조를 내 업무 환경(트렌드 레이더)에 맞게 갖추는 것이 이 도구의 목적이다.

## 2. 활용 시나리오 (누가, 언제 쓰나)

| # | 상황 | 사용 흐름 | 결과물 |
|---|---|---|---|
| S1 | 신규 아이템 검토 지시를 받았을 때 | 새 프로젝트 생성(주제·배경 입력) → **전체 자동 생성** → 어색한 단계만 메모 남기고 재생성 | 당일 공유 가능한 5단계 전략 초안 |
| S2 | 시나리오를 여러 방향으로 비교할 때 | 3단계(시나리오 플래닝)에 "규제 강화 시나리오 중심으로" 같은 메모 → 재생성 반복, 버전 이력으로 비교 | 시나리오별 대응 옵션 비교안 |
| S3 | 보고서 제출·정기 보고 | 최종 리포트 → 리포트 합성(경영 요약+액션 플랜) → 인쇄/PDF 또는 Markdown 다운로드 | 결재·공유용 리포트 1장 |
| S4 | 트렌드에서 기회를 발견했을 때 | 레이더에서 반응 급증 확인 → 그 키워드로 프로젝트 생성 → 각 단계에서 "트렌드 레이더 ↗"로 근거 재확인 | 트렌드 → 전략으로 이어지는 기록 |

## 3. 핵심 콘셉트

> **입력 한 번 → 5단계 자동 생성 → 보고서 한 장.**

벤치마크와 다른, 내 업무에 맞춘 3가지:

1. **트렌드 레이더 연동** — 프로젝트에 트렌드 키워드를 달아두면 모든 단계에서 원클릭으로 레이더 검색(근거 수집)으로 점프. 근거와 전략이 한 세트로 움직인다.
2. **컨텍스트 체이닝** — 각 단계 생성 시 프로젝트 정보 + **이전 단계 결과** + 단계 메모가 자동으로 프롬프트에 들어간다. 시장조사가 경쟁 분석의 기준이 되고, 시나리오가 아이디어 평가 기준이 되는 구조.
3. **서버 없는 개인 도구** — 정적 HTML 하나. API 키만 등록하면 브라우저가 직접 AI를 호출하고, 모든 데이터는 내 브라우저(localStorage)에만 남는다. 사내 배포·서버 승인 절차 없이 오늘부터 쓴다.

## 4. 화면 구성 (와꾸)

```
[프로젝트 목록] ─ 새 프로젝트 → [생성 폼] ─ 생성 → [5단계 워크스페이스] ─→ [최종 리포트]
      ↑ 검색·진행도(n/5)              │                 │  단계별: AI 생성/재생성        │  리포트 합성(요약+액션플랜)
      └── 트렌드 레이더 ←──────────────┴─ 키워드 연동 ────┘  메모·버전 이력·상태            └─ 인쇄/PDF·복사·Markdown
```

| 화면 | 구성 요소 | 역할 |
|---|---|---|
| ① 프로젝트 목록 | 검색, 새 프로젝트, 카드(키워드 태그·진행도 바 n/5) | 진입점. 진행 중인 검토 건 한눈에 |
| ② 생성 폼 | 기본 정보(제목*·산업·타깃 고객·타깃 지역·**트렌드 키워드**) + 전략 주제와 맥락(주제*·배경·회사 강점·비즈니스 모델 가설·제약 조건) | AI에게 줄 컨텍스트를 한 번에 수집 |
| ③ 5단계 워크스페이스 | 단계 카드 × 5: 목적 안내, **AI 생성/재생성**, 프롬프트 복사(키 없을 때 대안), 트렌드 레이더 점프, 결과 정리(수정 가능), 단계 메모, 상태(대기/진행 중/완료), 버전 이력(최근 10개 보관·복원) + 상단 **전체 자동 생성** | 실제 작업 공간. 생성→검토→수정→재생성 반복 |
| ④ 최종 리포트 | 전략 질문 자동 구성, **리포트 합성**(Executive Summary·핵심 인사이트 3·30/60/90일 액션 플랜), 단계별 결과 취합, 인쇄/PDF·전체 복사·Markdown 다운로드 | 산출물. 보고·결재·아카이빙 |

## 5. 5단계 프레임워크 정의

| 단계 | 목적 | 요구 산출물 | 다음 단계로 넘기는 것 |
|---|---|---|---|
| 1. 시장조사 | 시장 규모·성장 동력·미충족 수요 파악 | TAM/SAM/SOM, 성장 동인·저해 요인, 세그먼트별 JTBD, 트렌드×자사 강점 | 시장 인사이트 → 경쟁 비교 기준 |
| 2. 경쟁사 분석 | 경쟁 구도와 차별화 영역 도출 | 경쟁 매핑(직접/간접/대체), 벤치마킹 표, White Space 가설, 차별화 방향 | 경쟁 공백 → 시나리오 변수 |
| 3. 시나리오 플래닝 | 미래 변수별 대응 전략 준비 | 불확실성 축 2개, 2×2 매트릭스, 시나리오별 옵션·No-regret Moves | 시나리오 옵션 → 아이디어 평가 기준 |
| 4. 아이디어 발굴 | 실행 가능한 기회 도출·우선순위화 | 후보 5~6개, 평가 표(매력도/적합도/난이도), 상위 추천·선정 사유, MVP 실험 | 상위 아이디어 → 사업성 검토 대상 |
| 5. 사업성 검토 | Go/No-Go 판단 근거 확보 | 수익 모델·단위경제 가설, 진입 비용·핵심 자원, 리스크·완화, 최종 권고+90일 검증 과제 | 권고안 → 최종 리포트 액션 플랜 |

## 6. AI 동작 설계

- **단계 생성 프롬프트 구조**: `역할(신사업 전략 컨설턴트) + 프로젝트 정보 전체 + 이전 단계 결과(단계당 800자 요약) + 단계 메모(재생성 지시) + 단계 목적 + 요구 산출물 + 작성 원칙(근거 중심, 가설 명시)`
- **재생성**: 기존 결과는 버전 이력으로 자동 보관(최근 10개) 후 덮어씀. 메모에 지시를 남기고 재생성하는 것이 기본 반복 루프.
- **리포트 합성 프롬프트**: 5단계 결과 전체 → Executive Summary(5~7문장) + 핵심 인사이트 3 + 30/60/90일 액션 플랜.
- **제공자**: Claude(기본, `claude-sonnet-5`) / Gemini(`gemini-2.5-flash`, 무료 키 가능) 중 선택. 브라우저에서 직접 스트리밍 호출 — 결과가 화면에 실시간으로 써진다.

## 8. 운영 원칙 (중요)

1. **API 키는 개인 키** — 브라우저에만 저장되지만, 공용 PC에서는 등록하지 말 것.
2. **사내 기밀 입력 전 확인** — 외부 AI API로 전송되므로 회사의 생성형 AI 사용 정책 범위 안에서 입력할 것.
3. **AI 결과는 초안** — 수치·경쟁사 정보는 가설로 취급하고 레이더/원자료로 검증 후 보고에 사용할 것.
4. **백업은 Markdown** — localStorage는 브라우저 데이터 삭제 시 사라지므로, 중요한 프로젝트는 Markdown 다운로드로 보관할 것.

## 9. Quick Start

1. `strategy.html` 열기 (레이더 헤더의 "◆ 전략 프로젝트" 버튼)
2. ⚙ AI 설정 → Claude 또는 Gemini API 키 등록
3. 새 프로젝트 → 주제·배경 입력 → 생성
4. **전체 자동 생성** 클릭 → 5단계 초안 완성
5. 단계별로 메모 남기고 재생성 → 최종 리포트 → 리포트 합성 → 인쇄/PDF
